⚙️ Service Principal (Entra ID)

How to create the Service Principal and permissions
  1. Go to entra.microsoft.com → Identity → Applications → App registrations → New registration.
  2. Give it a name (e.g. M365-Forensics), single tenant, then register.
  3. Copy the Application (client) ID and the Directory (tenant) ID from the overview page.
  4. Under Certificates & secrets → New client secret, generate and copy the Value (shown only once).
  5. Under API permissions → Add a permission → Microsoft Graph → Application permissions, add:
    • AuditLog.Read.All — sign-in logs
    • Directory.Read.All — directory data
    • RoleManagement.Read.Directory — privileged roles
    • User.Read.All — user profile & photo
    • OrganizationalBranding.Read.All — (optional) company logo
  6. Click Grant admin consent for the tenant.
  7. Sign-in logs require an Entra ID P1/P2 license (Graph retains up to 30 days).

For 90+ days (Log Analytics): In Entra → Monitoring → Diagnostic settings, export SigninLogs to a Log Analytics workspace. Grant the Service Principal the Log Analytics Reader role on that workspace, then paste the Workspace ID above.
